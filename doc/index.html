Module columns

Columns - multiple column support in Pandoc's markdown.

This Lua filter provides support for multiple columns in latex and html outputs. For details, see README.md.

convert_explicit_columbreaks (elem)
Convert explicit columnbreaks. This function converts any explict columnbreak markup in an element into a single syntax: a Div with class columnbreak. Note: if there are column Divs in the element we keep them in case they harbour further formatting (e.g. html classes). However we remove their column class to avoid double-processing when column fields are nested.

Parameters:

  • elem Pandoc native Div element

Returns:

    elem modified as needed
tag_with_number_of_explicit_columnbreaks (elem)
Tag an element with the number of explicit columnbreaks it contains. Counts the number of epxlicit columnbreaks contained in an element and tags the element with a number_explicit_columnbreaks attribute. In the process columnbreaks are tagged with the class columnbreak_already_counted in order to avoid double-counting when multi-columns are nested.

Parameters:

  • elem Pandoc element (native Div element of class columns)

Returns:

    elem with the attribute number_explicit_columnbreaks set.

preprocess_columns (elem)
Pre-process a Div of class columns. Converts explicit column breaks into a unified syntax and count the Div's number of columns. When several columns are nested Pandoc will apply this filter to the innermost columns Div first; we use that feature to prevent double-counting.

Parameters:

  • elem Pandoc element to be processes (Div of class columns)

Returns:

    elem modified as needed
determine_column_count (elem)
Determine the number of column in a columns Div. Looks up two attributes in the Div: the user-specified columns-count and the filter-generated number_explicit_columnbreaks which is based on the number of explicit breaks specified. The final number of columns will be 2 or whichever of column-count and number_explicit_columnbreaks is the highest. This ensures there are enough columns for all explicit columnbreaks. This provides a single-column when the user specifies column-count = 1 and there are no explicit columnbreaks.

Parameters:

  • elem Pandoc element (Div of class columns) whose number of columns is to be determined.

Returns:

    number of columns (number, default 2).
